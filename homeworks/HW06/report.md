# Отчёт по домашнему заданию HW06 (S06): деревья решений и ансамбли

## 1. Информация о данных

**Выбранный датасет:** `S06-hw-dataset-01.csv`

**Задача:** бинарная классификация, целевая переменная — `target`.

**Размерность:**
- строк: **12 000**
- признаков (до предобработки): **30**
- технический столбец: `id` (удалён)

**Распределение классов (target):**
- класс 0: **67.66%**
- класс 1: **32.34%**
> Наблюдается умеренный дисбаланс (примерно 2:1), поэтому помимо accuracy использовались F1 и ROC-AUC.

**Типы признаков (по смыслу):**
- числовые: `num01`–`num24` (24 признака)
- категориальные-подобные (уже в виде целых кодов): `cat_contract`, `cat_region`, `cat_payment`
- целочисленный: `tenure_months`
- `id` — служебный, исключён из обучения

**Качество данных:**
- пропуски: **не обнаружены**
- данные синтетические (не описывают реальных людей/события)

---

## 2. Методы и подходы

### 2.1. Предобработка данных

**Загрузка и первичный анализ:**
- загрузка CSV по относительному пути;
- проверка типов столбцов и базовой статистики;
- анализ распределения `target`;
- визуализация дисбаланса классов.

**Очистка:**
- удалён столбец `id`;
- пропуски отсутствуют — импутация не требовалась;
- категориальные признаки представлены числовыми кодами, дополнительное кодирование не применялось.

**Подготовка выборок:**
- разбиение на `X` и `y`;
- **stratified** train/test split: **80/20**;
- фиксирован `random_state=42` для воспроизводимости.

---

### 2.2. Использованные модели

**Baseline:**
- `DummyClassifier(strategy="stratified")` — нижняя граница качества;
- `LogisticRegression` + `StandardScaler` — линейный baseline.

**Основные модели:**
- `DecisionTreeClassifier` — подбор гиперпараметров через `GridSearchCV`;
- `RandomForestClassifier` — ансамбль bagging + случайность по признакам;
- `GradientBoostingClassifier` — boosting-ансамбль, подбор параметров;
- `StackingClassifier` — ансамбль из нескольких моделей, метамодель: логистическая регрессия.

---

### 2.3. Метрики качества

Для сравнения моделей использовались:
- **Accuracy**
- **F1-score**
- **ROC-AUC**
- **Confusion Matrix** (визуализация ошибок)

> В силу дисбаланса классов основное внимание уделялось F1 и ROC-AUC.

---

### 2.4. Процедура эксперимента (честная постановка)

- **Train/Test split** фиксирован один раз (`random_state=42`).
- **Подбор гиперпараметров** выполнялся **только на train** через `GridSearchCV` (CV внутри train).
- **Финальная оценка** моделей проводилась **один раз на test**.
- Для лучшей модели выполнена интерпретация через **permutation importance**.
- Графики и результаты сохранялись в `artifacts/` (рекомендуемая структура: `artifacts/figures/`).

### 3.2. Лучшая модель

**Лучшая модель:** `StackingClassifier`

**Почему выбрана:** показала лучшие значения по всем ключевым метрикам (**Accuracy**, **F1**, **ROC-AUC**), что подтверждает преимущество ансамблирования разнородных моделей.

**Конфигурация (по итогам подбора):**
- базовые модели:
  - `LogisticRegression` (со `StandardScaler`)
  - `DecisionTreeClassifier(criterion="entropy", max_depth=7, min_samples_leaf=2, min_samples_split=10)`
  - `RandomForestClassifier(n_estimators=100, max_depth=15, max_features="log2", min_samples_split=2)`

**Качество на test:**
- Accuracy = **0.9275**
- F1-score = **0.8857**
- ROC-AUC = **0.9677**
