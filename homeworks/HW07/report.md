## 9. Выводы

### 9.1. Основные результаты
1. **Dataset-01:** KMeans показал лучшие результаты (Silhouette = 0.522) благодаря корректному масштабированию и сферической форме кластеров. DBSCAN нашёл 3 кластера без шума, но качество ниже.

2. **Dataset-02:** Оба алгоритма показали низкое качество, но KMeans (Silhouette = 0.307) незначительно превзошёл DBSCAN (0.085). Это неожиданно для датасета с нелинейной структурой, что может указывать на необходимость других методов или параметров.

3. **Dataset-03:** KMeans показал умеренное качество (Silhouette = 0.316), в то время как DBSCAN дал отрицательный результат (-0.104). Разная плотность кластеров создала проблемы для плотностного алгоритма.

### 9.2. Влияние препроцессинга
- Масштабирование оказалось критически важным для KMeans
- Выбросы не удалялись, что позволило DBSCAN корректно их идентифицировать как шум
- Отсутствие пропусков упростило подготовку данных

### 9.3. Рекомендации по выбору алгоритма
1. **Использовать KMeans, если:**
   - Кластеры предположительно сферические (как в dataset-01)
   - Все признаки имеют одинаковую важность
   - Требуется быстрая кластеризация

2. **Использовать DBSCAN, если:**
   - В данных присутствуют выбросы (dataset-02 показал 7.2% шума)
   - Неизвестно количество кластеров
   - **Но:** требует тщательного подбора eps, особенно для кластеров разной плотности

### 9.4. Сложности и решения
1. **Подбор k для KMeans:** Решено с помощью анализа silhouette score, оптимальные значения: 2, 2, 3 для датасетов 1-3 соответственно.

2. **Подбор eps для DBSCAN:** Использован k-distance график, но результаты показали, что подбор затруднён для данных со сложной структурой.

3. **Интерпретация метрик:** Silhouette Score оказался наиболее информативным, но низкие значения для dataset-02 и dataset-03 указывают на фундаментальные проблемы с разделимостью.

### 9.5. Заключение
Эксперимент показал, что даже при использовании внутренних метрик качества можно сделать обоснованный выбор алгоритма кластеризации. KMeans оказался более устойчивым выбором для данных датасетов, в то время как DBSCAN требовал тщательной настройки и показал нестабильные результаты. Для сложных структур данных (dataset-02 и dataset-03) можно рассмотреть другие методы, такие как иерархическая кластеризация или спектральная кластеризация.
