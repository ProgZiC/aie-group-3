## 2. Структура проекта

Проект организован в следующей структуре:

- `requirements.txt` – зависимости проекта (библиотеки Python, необходимые для запуска).
- `report.md` – отчёт по проекту (постановка задачи, данные, эксперименты, результаты).
- `self-checklist.md` – чеклист самопроверки проекта перед сдачей.
- `notebooks/` – экспериментальные ноутбуки:
  - `01_eda.ipynb` – разведочный анализ (EDA) и предобработка данных.
  - `02_models.ipynb` – обучение baseline, тюнинг CatBoost (Optuna) и анализ интерпретируемости (SHAP).
- `src/` – основной код проекта:
  - `service.py` – исходный код REST API сервиса на FastAPI.
- `data/` – демонстрационные данные:
  - `telco_cleaned.csv` – очищенный датасет, генерируемый на этапе EDA.
- `artifacts/` – сохранённые модели и препроцессоры:
  - `catboost_churn_model.cbm` – веса финальной модели.
  - `scaler.pkl` и `features.pkl` – артефакты для правильной обработки входящих JSON-запросов.
- `.env.example` – шаблон конфигурационного файла.
- `Dockerfile` – инструкции для сборки Docker-образа сервиса.

### 3.2. Установка окружения (Локальный запуск)


## 4. Как запустить проект

Ниже описаны основные сценарии запуска: для воспроизведения экспериментов и для поднятия API-сервиса.

### 4.1. Воспроизведение экспериментов и обучения

```bash
# Перейдите в корень проекта
cd project
```

# Запустите Jupyter Notebook
В открывшемся интерфейсе поочередно запустите ячейки в ноутбуках:

- `notebooks/01_eda.ipynb` (скачивание и подготовка данных).

- `notebooks/02_models.ipynb` (обучение модели и генерация артефактов).

### 4.2. Автоматический запуск сервиса (API)

Для Windows:
Просто запустите двойным кликом файл `start.bat` в корне проекта (или выполните в терминале `.\start.bat`).

Для Linux / macOS:
Дайте скрипту права на исполнение и запустите его:
```bash
chmod +x start.sh
./start.sh
```
Описание работы сервиса:

Сервис поднимается на порту 8000.

Реализованы эндпоинты:

- GET /health — проверка работоспособности (Liveness probe).

- POST /predict — основной эндпоинт для предсказания оттока.

Как протестировать: Откройте в браузере http://localhost:8000/. Сработает автоматический редирект на интерфейс Swagger UI (/docs), где можно отправить тестовый JSON-запрос и получить предсказание.

В текущей реализации проекта автоматические тесты (например, pytest) не настроены.

## 8. Ограничения и дальнейшая работа
Текущие ограничения проекта:

- Модель работает со «статичным» срезом данных и не учитывает историю поведения абонента во времени (отсутствует анализ временных рядов).

- Сервис является stateless: история предсказаний не сохраняется между перезапусками приложения.

Направления для дальнейшего развития:

- Интеграция базы данных (например, PostgreSQL) для сохранения истории запросов и ответов модели.

- Настройка инструментов мониторинга (например, Evidently AI) для отслеживания Data Drift.

    Внедрение A/B тестирования на уровне API для маршрутизации трафика между разными версиями моделей.
